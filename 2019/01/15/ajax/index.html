Ajax、Fetch以及Axios

发表于 2019-01-15 | 分类于 web前端

什么是Ajax

AJAX（Asynchronous JavaScript and XML），指的是通过 JavaScript 的异步通信，从服务器获取 XML 文档从中提取数据，再更新当前网页的对应部分，而不用刷新整个网页

简单的来说Ajax就是用JS向服务端发送请求，然后可以得到服务器响应的内容

原生Ajax

AJAX技术的核心环节：XMLHttpRequest对象

XMLHttpRequest对象是浏览器提供的一个API，用来顺畅地向服务器发送请求并解析服务器响应，当然整个过程中，浏览器页面不会被刷新

实现步骤：

  1. 创建XMLHttpRequest实例
  2. 打开请求: 发送请求之前的一些配置项
  3. 事件监听：一般监听的都是onreadystatechange事件
  4. 发送请求

readyState 代理当前所处的状态：
0 代理被创建，但未调用 open()方法
1 open() 被调用，send() 未调用
2 send() 被调用，且头部和状态已经可获得
3 响应体下载中；responseText 中已获取部分数据
4 下载操作已完成

jQuery中Ajax

JQuery的Ajax是对原生XHR的封装，并且还支持JSONP，非常的方便，用的人也特别多，不过随着Vue，React等框架的流行，如果我们想使用JQuery的Ajax，就得引入整个jQuery，未压缩的有266kb，压缩以后也有86kb，显然有点小题大做，非常不合理，于是就有了Fetch

Fetch

fetch号称是ajax的替代品，它的API是基于Promise设计的

fetch是比较底层的API，相对于jQuery Ajax来说,很多情况下都需要我们再次封装，在使用上自然没有他们方便，而且还有一个写缺点

Fetch的缺点：

  • 只对网络请求报错，对400，500都当作成功的请求
  • Fetch 请求默认是不带 cookie 的，需要设置 fetch(url, {credentials: 'include'})

下一代的HTTP库：Axios

Axios本质上也是对原生XHR的封装，只不过它是Promise的实现版本，并且Vue作者尤雨溪推荐大家用axios替换JQuery ajax

它有以下几大特性：

  • 可以在node.js中使用
  • 提供了并发请求的接口
  • 支持Promise API
